System Enhancements

The following enhancements were made to the system.

WIRIS MathML Editor Added to Content Editors

A new and improved MathML Editor was added to content editors for items, passages, rubrics, interactions, directions and footnotes. This replaces the fMath Editor that previously existed in the content editors. You can access it from the new Math Editor icon  added to the ribbon bar of these content editors.

Figure 1 Accessing the WIRIS MathML Editor

You can use the new WIRIS Editor to create and edit new MathML equations, edit equations created with the previous fMath Editor, and edit equations imported into the system.

You can export MathML equations created with the WIRIS Editor as well as continue to export those created with the previous fMath Editor and those imported into the system.

For more information on using the WIRIS MathML Editor, click Manual at the bottom-right corner of the editor.

For more information on LaTex and how to enter it into the WIRIS MathML Editor, click LaTex at the bottom-right corner of the editor.

Figure 2 Accessing Documentation on WIRIS and LaTex

 

See also: “Appendix C: Using the MathML Equation Editor" in the ACT CDE User Guide.

TestNav8 Parameters Added to Forms Manager

Several parameters needed in TestNav8 related to forms and form parts now appear on their respective pages in the Forms Manager.

For form parts, the following parameters are now available on the Create Form Part and Edit Form Part pages:

  • Section Title – (Text Field) Text presented to test-takers at the start of a new section. Required for all form parts. Character limit of 50. Values accepted include alphanumeric, space, underscore, and hyphen.
  • Allow Reentry – (Checkbox) Allows test-takers to exit and reenter a test. Defaults to True.
  • Must Answer to Continue (Checkbox) Requires test-takers to submit an answer before continuing on with the test. Defaults to False.
  • Seal Code – (Checkbox) Requires test-takers to enter a four-digit code to continue to the next section. Defaults to False.

Figure 3 Additional TestNav8 Parameters for Form Parts

For forms, the following parameters are now available on the Create Form and Edit Form pages:

  • Exit Enabled (Checkbox) Defaults to True.
  • Save Responses – (Checkbox) Saves the responses to each question. Defaults to True.
  • Secure – (Checkbox) Defaults to True.

Figure 4 Additional TestNav8 Parameters for Forms

Content Specialists and users with admin permissions can configure these parameters by accessing the respective pages. The values you enter export as needed to TestNav8.

Option to Append Form ID to Items and Passages Added for Form Export

When creating or editing a form, content specialists and users with admin permissions now have the option to append the Form ID to item and passage identifiers when you export the form package.

More specifically, the system adds the Form ID to the following locations in the form package:

  • XML and CSS Filenames for Items and Packages
    Example: I00033_3774_E34I011.xml
  • Identifier and HREF Parameters in xml
    Example: <assessmentItemRef href="I00033_3776_E34I011.xml" identifier="I00033_3776_E34I011"/>
  • Identifier Parameters in Discreet Item XML Files
    Example: <assessmentItem identifier="I00033_3738_E34I011">
  • Identifier and HREF Parameters in Item XML Files Associated with a Passage
    Example: <assessmentItem identifier="I00033_3738_E34I011"> <xi:include parse="xml" href="p1220_E34I011.xml"/>
  • Identifier and HREF Parameters in imamanifest.xml
    Example: <resource identifier="I00033_3779_E34I011" href="I00033_3779_E34I011.xml"> <file href="I00033_3779_E34I011.xml" /> <dependency identifierref="I00033_3779_E34I011.css" />

By appending the Form ID, you ensure TestNav8 will not overwrite other versions of the items and passages when you upload the package to TestNav8 because they’ll be unique to your form.

To enable this functionality, select the Add Form ID to Identifier option on the Create Form or Edit Form page.

Figure 5 Configuring a Form to Append the Form ID to Identifiers

Export of Extended Text Entry Items Improved

When you export a form with an item that has a single extended text entry interactions, the system now exports the responseProcessing logic changes to the item XML

When you export a form with an item that has multiple extended text entry interactions, the system now ensures the following occurs to the item XML:

  • Includes a responseProcessing element
  • Includes a separate Outcome Value for each score
  • Lists each response on a separate line

These export changes ensure the TestNav8 import mechanism can import the items successfully, score each part of the item accurately, and then sum the scores to produce a final score for the item.

PBT and CBT Styles Added to Content Editors

Content Specialists and Item Writers can now access and apply either a PBT (Paper-Based Testing) or CBT (Computer-Based Testing) style to item, passage, and interaction content. The formatting styles appear on the Ribbon Bar of the Create Item page, Create Passage page, Edit Item Content panel, Edit Passage Content panel, and Edit Item Interaction panel.

When you apply these styles to selected text within a item/passage/interaction, the appearance of the text does not change in the content editor, but it does affect how the text appears in TestNav8 and content previewers.

  • PBT
    Ensures only the selected text exports as part of a paper-based test when exporting the item using Export to PBT.
  • CBT
    Ensures only the selected text exports as part of an online test when exporting the item using Export to TestNav8.

Note: Text designated with PBT will not display in the TestNav8 Previewer or system previewers (such as the Item Preview Panel or Create Passage Preview tab). Text designated with CBT will continue to display.

Figure 6 PBT and CBT Styles on the Edit Passage Content Panel

Users can apply further styles to the same text, such as bold or italics, while maintaining the styles listed above. The formatting remains intact upon export.

VITA Block Styles for Passages Added to Passage Editors

Content Specialists and Item Writers can now access and apply three new VITA Block Styles to passage content. The formatting style appears under Formats > VITA Block Styles on the Create Passage page, and Edit Passage Content panel.

  • Advanced Organizer
    When viewing the passage in TestNav8, the text formatted with the Advanced Organizer style appears outside and above the scrollable box instead of appearing inside it. This content commonly includes text that introduces the passage topic and, more importantly, illustrates the relationship between the passage and information they already know.
  • Passage Head
    Similar to Advanced Organizer, when viewing the passage in TestNav8, the text formatted with the Passage Head style appears outside and above the scrollable box instead of appearing inside it. This content commonly includes a small amount of text that only introduces the passage topic.
  • Stimulus Paragraph Number
    Applying Stimulus Paragraph Number to content centers the text in the editor. This content typically includes numbers before each paragraph in a passage. When exported, the style changes to “text-align: center” in order to comply with TestNav8. This style is also available on the Edit Reference Passage Content panel.

When you apply Advanced Organizer or Passage Head to selected text within a passage, the appearance of the text does not change in the content editor, but it does affect how the text appears in TestNav8 and the TestNav8 Previewer.

Figure 7 Advanced Organizer Applied to Passage Content

Users can apply further styles to the same text, such as bold or italics, while maintaining the styles listed above. The formatting remains intact upon export of the passage.

VITA Block Styles for Items Added to Item Editors

Content Specialists and Item Writers can now access and apply three new VITA Block Styles to item content. The styles appear under Formats > VITA Block Styles on the Create Item page and Edit Item Content panel.

  • Boxed Directions Stem
    When you use the Paired Passage Instruction style to create a scrollable box around item content, the text formatted with the Boxed Direction Stem style appears outside and directly below the scrollable box. The selected text should include text that applies to all interactions in the item.
  • Paired Passage Instruction
    When you apply this style to item content, it creates a scrollable box around content not formatted with this style and places the text formatted with this style outside and directly above the scrollable box. This style requires you to use the Prompt style on the stem of the first interaction or else that stem will appear within the scrollable box.
  • Prompt
    When you use the Paired Passage Instruction style to create a scrollable box around item content, the text formatted with this style appears outside and below the scrollable box (similar to the Boxed Directions Stem), except the selected text should include text that applies to only one interaction for the item. You can apply this style to the stem for all interactions in an item.

Figure 8 VITA Block Styles Applied to Item Content

Users can apply further styles to the same text, such as bold or italics, while maintaining the styles listed above. The formatting remains intact upon export of the passage.

VITA Inline Styles Added to Content Editors

Content Specialists and Item Writers can now access and apply two new VITA Inline Styles to any content, including passages, items, and interactions. The styles appear under Formats > VITA Inline Styles on the Create Item page, Create Passage page, Edit Item Content panel, Edit Interactions panel, Edit Passage Content panel, and Edit Reference Passage Content panel.

  • Boxed Directions Stem
    When you use the Paired Passage Instruction style to create a scrollable box around item content, the text formatted with the Boxed Direction Stem style appears outside and directly below the scrollable box. The selected text should include text that applies to all interactions in the item.
  • Inline No-Break
    When you apply this style to content, the text appears with an orange highlight all characters appear on a single line. When exported, the style changes to “white-space:nowrap” in order to comply with TestNav8 and no longer includes the orange highlighting.

Users can apply further styles to the same text, such as bold or italics, while maintaining the styles listed above. The formatting remains intact upon export of the passage.

Passage Underline Style Updated to Comply with Remove Incorrect Tags

Content Specialists and Item Writers can now apply the Passage Underline style to content tagged with the Dynamic Text Remove Incorrect tag. The formatting style appears under Formats > VITA Inline Styles on the Create Passage page, Edit Passage Content panel, and Edit Reference Passage Content panel.

When you apply this style to text within the Remove Incorrect tag, the text appears underline, but it does affect how the text appears in TestNav8 and the TestNav8 Previewer.

Users can apply further styles to the same text, such as bold or italics, while maintaining the styles listed above. The formatting remains intact upon export of the passage.

Gray Well Removed for Items Exported to TestNav8

When you used the Preview in TestNav8 or Export to TestNav8 options, item content appeared within a gray well. That gray well no longer appears in the preview and is no longer exported as part of the item XML. The content is embedded directly on the test delivery space as show on the right side of the image below. The passage continues to appear in its own scrollable box as show on the left side of the image.

All users can access the Preview in TestNav8 option from the Preview tab of the Create Item page. Content Specialists and users with administrator permissions access the Export to TestNav8 option from the Manage Test Form page.

 

 

Performance of Item Reports Improved

Item reports perform faster due to changes we made to how the reports pull in data.

Specifically, instead of having the system retrieve all records every time, the SQL construction now depends on your filter selection. Also, a paging feature was added to HTML reports so the reports only have to retrieve a small chunk of data with each page instead of the entire report.

Item Set Workspace Enhancements

The following enhancements were made to the Item Set Workspace.

Note: The Item Set Workspace is currently restricted to Content Specialists.

Uploading Print-Ready Images for Passage Images

You can now upload a print-ready image (EPS file) for an already uploaded image for a passage using a new icon added to the Manage Passage Assets panel in the Item Set Workspace. The icon offers the same functionality as the Insert Image icon in the Content Editor the Create Passage page.

Figure 11 Uploading a Print-Ready Image File

The system uses the print-ready version of the image in the delivery of printed tests. You can only upload one print-ready EPS file per image file for a passage. The maximum files size for EPS images is 16 MB.

See also: “Add Image Files” in the “Edit the Passage” section of the ACT CDE Item Set Workspace User Guide.

Application Changes

Preview Environment Created

A new PREVIEW environment now exists for ACT CDE. This is in addition to our QA, UAT, and PROD environments.

The PREVIEW environment is restricted to a small number of users responsible for testing and approving new features. It’s a place where users can preview a new feature before committing it to the system or troubleshoot an existing issue without compromising live data.
